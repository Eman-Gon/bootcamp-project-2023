Building DepScope: A Multi-Agent Dependency Analyzer

Feb 2025 · Continual Learning Hackathon · 1st Place

At the Continual Learning Hackathon, our team set out to tackle a problem every developer faces but rarely thinks about: how risky are the open-source dependencies your project relies on? The result was DepScope, a multi-agent platform that evaluates dependencies for security vulnerabilities, maintenance health, and bus factor in real time.

The Problem

Modern software projects pull in dozens, sometimes hundreds, of open-source packages. Each one is a potential attack vector or a ticking time bomb if the maintainer walks away. Existing tools like Dependabot flag known CVEs, but they miss the bigger picture — is the project actually maintained? Is it a one-person operation? How quickly do they patch issues?

The Architecture

We designed DepScope around three parallel AI agents, each focused on a different risk dimension:

  • Security Agent — Scans for known vulnerabilities, analyzes dependency trees for transitive risks, and checks for signs of supply chain attacks.
  • Maintenance Agent — Evaluates commit frequency, issue response times, release cadence, and documentation quality.
  • Bus Factor Agent — Analyzes contributor distribution to flag single-maintainer risks and assess community health.

All three agents run concurrently, orchestrated through Composio. Results stream to the frontend via Server-Sent Events so you see findings as they come in rather than waiting for everything to finish.

Real-Time Alerts

For critical findings — like an actively exploited CVE or a key dependency with a single maintainer who hasn't committed in months — we integrated Plivo voice alerts. The system can call you and read out the critical risk summary so nothing gets buried in a notification queue.

Takeaways

The biggest lesson was how well multi-agent architectures parallelize analysis work. Instead of one monolithic LLM prompt trying to evaluate everything at once, splitting concerns across specialized agents gave us better accuracy and faster results. It also made the system modular — adding a fourth agent for, say, license compliance would be straightforward.

Winning 1st place was a great validation that this kind of developer tooling resonates. Dependency risk isn't glamorous, but it matters, and AI agents are a natural fit for automating the analysis.
